#### Rationale

**Why this decision:**

**Completeness:**
- System maintains maximum ontological precision (390k concepts)
- No compromise on theoretical completeness
- Suitable for AI, scientific databases, formal ontology

**Accessibility:**
- Learners focus on manageable Essential Core (78k concepts)
- Clear learning path without intimidation
- 95% of communication needs met with essential tier

**Elegance:**
- Clean mathematical structure (diagonal subset)
- Visually obvious distinction (vowel patterns)
- Precedent in other systems (basic vs. advanced tiers)

**Practicality:**
- Essential Core documentable in 3-5 years
- Professional Expansion developed progressively
- Community can contribute specialized domains

**Use Cases:**
- Casual conversation: Essential Core
- Literature, journalism: Essential Core
- Scientific papers: Professional Expansion
- AI/NLP systems: Professional Expansion (full precision)
- Database ontologies: Professional Expansion

**Major architectural refinement (v0.2.0):**

The dual-tier system represents a significant evolution in the project's design philosophy. By separating Essential Core (for humans) from Professional Expansion (for specialists/AI), we achieve:

1. **Ambitious completeness** without sacrificing accessibility
2. **Clear learning path** for newcomers
3. **Maximum precision** for technical applications
4. **Realistic development timeline** (Essential Core prioritized)
5. **Elegant mathematical structure** (diagonal subset pattern)

This decision positions the language as both a practical communication tool AND a comprehensive ontological framework — serving casual speakers and AI systems alike without forcing either to compromise.
