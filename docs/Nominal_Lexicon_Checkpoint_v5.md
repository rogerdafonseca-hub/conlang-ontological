## 1. DUAL-LAYER ARCHITECTURE

The system has two synchronized layers:

| Layer | Name | Format | Function |
|---|---|---|---|
| Deep | Sistema Ontológico | VCV + CV + CV + CV... | Semantic infrastructure. No naming. Defines what something IS structurally. |
| Surface | Léxico Nominal | CVCV + CV | Human interface. Names real things. Short, usable, anchored in the world. |

**Key axiom:** The word does not carry the structure — it POINTS to it. Every word is a projection of an ontological vector. The relationship is bidirectional: word → vector, vector → word.

**Analogy:** The léxico is the view from outside the tree (leaves first). The sistema ontológico is the tree's internal structure (root to branch to tip).

## 2. SISTEMA ONTOLÓGICO

### Format
VCV + CV + CV + CV...

### VCV opener
A mode marker — signals "you are in the ontological system", same principle as numerals. May receive semantic meaning later. Currently functions as a cluster identifier only.

### Scale
Generalidade → Especificidade (A→U on both vowel axes)

### Vowel scale — universal rule

| Vowel | Position |
|---|---|
| A | most general / broadest |
| E | general-medium |
| I | neutral center |
| O | specific-medium |
| U | most specific / most particular |

- **V₁** = position of the field within the domain
- **V₂** = position of the word within the field

## 3. LÉXICO NOMINAL

### Format
CVCV + CV (extendable)

### Scale
Frequência/importância → Raridade/tecnicidade

| Length | Use |
|---|---|
| CVCV | Universal core — most frequent and important concepts |
| CVCVCV | Useful expansion — important but less daily |
| CVCVCVCV+ | Technical / specialized / rare |

### Projection Rules

| Element | Rule | Notes |
|---|---|---|
| C₁ | MANDATORY | Always mirrors the primary domain. P-words = matter, N-words = living, etc. |
| C₂ | PREFERENTIAL | Soft semantic clustering within domain. Can be overridden for phonetics or collision avoidance. |
| Vowels | FREE | Purely distinctive. No semantic hierarchy in léxico. |
| Word length | FREQUENCY | Shorter = more common/important. |

## 4. UNIVERSAL ONTOLOGICAL RULE

> A domain never contains: composition, interaction, or contextual instance.

These always go to field combinations. Specifically:
- No specific events in V → instances always involve T
- No group-in-action in K → action goes to V+K
- No emotion caused-by-X in W → causation goes to X+W

## 5. SEMANTIC AXIS LAW

> The horizontal axis (C₂) defines OPERATIONS, not concepts.

### Axis Constraints

- **C₁ (vertical axis):** Defines ontological progression of the domain
- **C₂ (horizontal axis):** Defines operational transformation applied to the domain

C₂ must be: domain-independent, reusable, operation-based (not concept-based), semantically consistent.

**Interpretation rule:** Meaning = Ontology (C₁) × Operation (C₂)

### Finalized C₂ Axes

## 8. OPERATIONAL BOUNDARIES

### M vs W
| Example | Domain |
|---|---|
| Thinking, believing, remembering | M |
| Joy, fear, sadness | W |
| Nostalgia, anxiety | M+W |
| Intention | M (with possible W) |

Rule: W never structures thought — it only modulates. M can exist without W. W never exists without some mental state base.

### B vs Y
| Example | Domain |
|---|---|
| Sound, gesture, graphic symbol | B |
| Datum, fact, concept | Y |
| Physical text (paper with ink) | P+B |
| Knowledge in mind | M+Y |
| Silent thought | Y (no B needed) |
| Noise without meaning | B (no Y needed) |

Rule: B is the vehicle. Y is the content. They can exist independently.

### J̌ vs V+G

| Criterion | J̌ | V+G |
|---|---|---|
| Ontological type | Property | Process |
| Temporal anchoring | Atemporal / general | Temporal / eventive |
| Nature | Stable pattern of motion | Change of position as event |
| Example | Orbit, current (as concept) | Moving, flowing (as event) |

### V reification
Rule: V contains process types, not instances. "Running" (type) = V. "Yesterday's race" = T+V. V becomes a verb only in predicative position.

## 10. GRAMMAR — FAMILY V (REVISED)

V-family is reformulated as **activation and derivation operators**, not verbal categories.

### CV — Function Activators (pre-word, change the grammatical function of the following word)

| Particle | Function | Example |
|---|---|---|
| **va** | activates as verb/action (omissible in obvious context) | *va wa be* = to love you / *wa be* = love you |
| **ve** | activates as adjective/quality | *ve para* = watery |
| **vi** | activates as instrument | *vi pata* = that uses stone |
| **vo** | activates as collective/set | *vo pata* = quarry |
| **vu** | negation of the following word | *vu wa* = lack of love / not-love |

### VC — Derivational Operators (pre-word, derive new word class)

| Particle | Function | Equivalent | Example |
|---|---|---|---|
| **av** | agent / doer | -er, -or | *av vapa* = runner |
| **ev** | having/causing the quality | -ous, -ful, -ish | *ev wa* = loving |
| **iv** | result / product | -ed, past participle | *iv vata* = the broken (thing) |
| **ov** | iterative / does repeatedly | habitual agent | *ov vapa* = one who runs constantly |
| **uv** | reciprocal / one another | mutual | *uv wa* = mutual love |

**Key rule:** All V-family particles precede the target word. They are never suffixed.

## 11. GRAMMAR — FAMILY W (REVISED)

W-family reformulated as **primary emotional scale** — direct, usable words for emotional states.

**Note:** W uses inverted gradation — a = positive, u = negative.

### CV System

| Particle | Function | Affective value |
|---|---|---|
| **wa** | love | Positive (strong) |
| **we** | like | Positive (mild) |
| **wi** | neutral / indifferent | Neutral |
| **wo** | dislike | Negative (mild) |
| **wu** | hate | Negative (strong) |

**Usage example:** *ba va wa be* = I love you (full) / *wa be* = love you (colloquial)

## 12. GRAMMAR — FAMILY H (REVISED)

H freed from "Relation" and reassigned to **Expression/Interjection**. H-family handles *how* something is said, not *what*.

**Note on interrogation:** K-family = reference words (who, what, where...) usable in any context. H-family marks interrogative *mode*. To ask a question: K-word + *ha*. Example: *ke ha* = "who?" as question. *ke* alone = "who" in declarative context.

### CV — Primary Expressions

| Particle | Function |
|---|---|
| **ha** | interrogative mode marker |
| **he** | confirmation / agreement ("right?", "indeed") |
| **hi** | topic marker ("as for X...", "regarding X...") |
| **ho** | surprise / exclamation |
| **hu** | hesitation / pause filler ("hm", "uh") |

### VC — Secondary Expressions

| Particle | Function |
|---|---|
| **ah** | emphasis / focus ("it is X that...") |
| **eh** | doubt / uncertainty |
| **ih** | indecision / conflict (?!) |
| **oh** | citation marker (before = introduces; after = attributes) |
| **uh** | emphatic negation |

### VCV — Nuanced Expressions

| Particle | Function |
|---|---|
| aha | rhetorical question |
| ehe | irony / sarcasm |
| ihi | relief |
| oho | deep admiration / wonder |
| uhu | pain / discomfort |
| ahu | vocative / direct address ("O X!") |
| ehi | mild disagreement |
| oha | positive surprise |
| uhe | lament / sorrow |
| iho | contrast marker ("but X...") |

## 13. GRAMMAR — OTHER CHANGES (v0.7.1)

### Family M — VC removed
VC of M is obsolete. Negated modals use *vu* (V-family negation): *vu ma* = cannot, *vu me* = may not. VCV covers conditional and nuanced modal gradations.

### Family S — *us* removed
*us* (at exact point) was redundant with *su* (exactly, precisely). Only *su* retained.

### EC TR — Deactivated
TR (Movement/Direction) deactivated. Movement is fully covered by X/Y/Z spatial system combined with G (aspect) and P (preposition). TR slots reserved for potential future repurposing.

## 14. VERBAL SYSTEM

### va as verbal activator
*va* transforms any following word into a verbal predicate. It is omissible when predicative context is obvious.

| Form | Register |
|---|---|
| *ba va wa be* | formal / explicit — "I love you" |
| *ba wa be* | subject present, va omitted |
| *wa be* | colloquial / obvious context — "love you" |

### Minimum Syntax

| Element | Rule |
|---|---|
| Basic order | SVO |
| Predicate | V-root or any word in post-subject position with *va* |
| Nominal reading | Word outside predicative position |
| Negation | *vu* before the target word |
| T (tense) | Optional operator — not required for predication |
