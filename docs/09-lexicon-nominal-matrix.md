# 📘 NOMINAL LEXICON MATRIX

**Ontological Organization and Functional Modifiers**

_Version 0.6.5 — Definitive Domain Structure_

---

## 🎯 Overview

The nominal lexicon is organized by **ontological domains** using the 20 base consonants + 7 defined consonant clusters. Each domain contains systematically derived words that can be modified using the 24 functional modifiers.

**Core Principle:**
- **CV¹** = Ontological domain (broad category)
- **CV²** = Subcategory (specific type within domain)
- **Vowel gradation** = Systematic variation within subcategory
- **CV(final)** = Functional modifier (ALWAYS last syllable, ONE per word maximum)

**NEW in v0.6.5:**
- **Definitive domain order:** 20 immutable base domains (P→T→K→B→M→G→V→R→J̌→S→F→H→N→D→L→Ľ→W→X→Y→Z)
- **Exception group:** Positions 17-20 (W, X, Y, Z)
- **Dual-function X, Y, Z:** Different domains in grammar vs. lexicon

## 📐 WORD STRUCTURE RULES

### **Basic Structure:**

```
[Domain][Subcategory][...Additional Syllables...][MODIFIER]
  CV¹       CV²              CVⁿ                    CV(final)
```

### **Position of Modifier:**

- **2-syllable base** → modifier is 3rd syllable
- **3-syllable base** → modifier is 4th syllable
- **4-syllable base** → modifier is 5th syllable
- etc.

## 🎯 DESIGN PRINCIPLES

### **What IS marked:**

✅ **Superlative/Comparative:** fo (more/-er), fu (most/-est) via F family  
✅ **Cases:** Full set via P (preposition) family CV/VC  
✅ **Plural/Number:** N (quantity) family or K modifier  
✅ **Diminutive/Augmentative:** M modifier (magnitude)  
✅ **Tense/Aspect:** T (tense), G (aspect) families  
✅ **Evidentiality:** Ľ family (hearsay → witnessed)  
✅ **Causality:** X domain + J̌ modifier combinations
✅ **Epistemic stance:** Y domain (knowledge) + Ľ modifier
✅ **Intensity marking:** Z domain + P/GR modifiers

### **What is NOT marked (intentional gaps):**

❌ **Definite/Indefinite articles:** Use demonstratives (di = this ≈ the)  
❌ **Grammatical gender:** System is genderless by design  
❌ **Animacy marking:** Not marked (optional semantic feature)

## 🔄 WORD FORMATION WORKFLOW

### **Step 1: Choose Domain (CV¹)**
Identify which of the 27 domains best fits your concept
- Positions 1-16: Standard gradation
- Positions 17-20: Exception group (check gradation rules)
- Positions 21-27: Specialized ECs

### **Step 2: Choose Series (CV²)**
Select the appropriate subcategory within the domain
- Each domain has 5 series (a, e, i, o, u)
- Each series represents a distinct semantic axis

### **Step 3: Choose Gradation (Final Vowel of CV²)**
Place your concept on the gradation scale
- Standard: a (minimum) → e → i → o → u (maximum)
- Exception W: a (positive) → u (negative)
- Check domain-specific gradation principles

### **Step 4: Add Modifier (Optional)**
Attach ONE modifier at the end if needed
- Choose from 24 modifiers
- Select appropriate vowel (a→e→i→o→u)
- Modifier ALWAYS comes last

3. **Focus on high-frequency series** first
