# 📘 NOMINAL LEXICON MATRIX

**Ontological Organization, Domain Matrices, and Word Formation**

_Version 0.7.0 — Updated Ontological Structure_

---

## 🎯 Overview

The nominal lexicon operates on two synchronized layers. The deep layer — the **Sistema Ontológico** — defines the structural address of every concept. The surface layer — the **Léxico Nominal** — provides the short, usable words that humans actually speak.

**Key axiom:** The word does not carry the structure — it POINTS to it. Every word is a projection of an ontological vector. The relationship is bidirectional.

---

## 🌐 DUAL-LAYER SYSTEM

### Layer A — Sistema Ontológico

**Format:** VCV + CV + CV + CV...
**Scale:** Generalidade → Especificidade (A→U on both vowel axes)
**Function:** Semantic infrastructure. Defines structural address. Does not produce spoken words directly.

| Position | Unit | Vowel scale |
|---|---|---|
| VCV | Domain marker | — |
| CV₁ | Field (A=most general → U=most specific) | A→U |
| CV₂ | Area | A→U |
| CV₃+ | Subnucleus | A→U |

### Layer B — Léxico Nominal

**Format:** CVCV + CV (extendable)
**Scale:** Frequência/importância → Raridade/tecnicidade

| Length | Use |
|---|---|
| CVCV | Universal core — most frequent and important |
| CVCVCV | Useful expansion — less daily |
| CVCVCVCV+ | Technical / specialized / rare |

### Projection Rules

| Element     | Rule                                                                                       |
| ----------- | ------------------------------------------------------------------------------------------ |
| C₁          | MANDATORY — mirrors primary domain (P-words = matter, N-words = living, etc.)              |
| C₂          | PREFERENTIAL — soft semantic clustering. Overridable for phonetics or collision avoidance. |
| Vowels      | FREE — purely distinctive. No semantic hierarchy in lexicon.                               |
| Word length | FREQUENCY — shorter = more common/important.                                               |

## 📐 WORD STRUCTURE

```
[Domain C₁][V₁][Field C₂][V₂]  +  [CV modifier — always last, max one]
      CVCV base word                  optional functional modifier
```

- **2-syllable base** → modifier is 3rd syllable
- **3-syllable base** → modifier is 4th syllable
- **4-syllable base** → modifier is 5th syllable

## 🧱 MATRIX PATTERN — UNIVERSAL STANDARD

Every domain nucleus (same-domain pair, e.g. PP, MM, KK) follows a universal 5×5 matrix:

### Vertical Axis (V₁) — Ontological type progression

| Row | Function |
|---|---|
| -A | Structural definition — most fundamental concept |
| -E | Fundamental unit — discrete entities |
| -I | Applied state / form — how entities manifest |
| -O | Aggregation / organization — how entities combine |
| -U | Properties / abstractions — measurable or emergent qualities |

### Horizontal Axis (V₂) — Internal refinement

| Column | Role |
|---|---|
| A- | Base concept |
| E- | Unit / refinement |
| I- | Internal structure |
| O- | Organization / combination |
| U- | Specialization / definition |

### Three validation tests

1. **Same type:** "Is this still the SAME ontological type, at a different level?" If no → wrong row.
2. **Refinement:** "Is this a refinement of the same base concept?" If no → wrong column.
3. **Purity:** "Does this involve use, action, or context?" If yes → does not belong in a pure nucleus.

## 🔀 KEY OPERATIONAL DISTINCTIONS

### J̌ vs V+G

| Criterion | J̌ | V+G |
|---|---|---|
| Type | Property | Process |
| Temporal | Atemporal | Eventive |
| Nature | Stable pattern of motion | Change of position as event |
| Example | Orbit, current (as concept) | Moving, flowing (as event) |

### M vs W

M = cognitive structure. W = affective valence. Hybrid states (nostalgia, anxiety) = M+W. W never structures — it only modulates.

### B vs Y

B = signal vehicle. Y = informational content. Independent — B can exist without Y (noise), Y without B (silent thought).

### V reification

V in léxico = process type (reified). "Running" as concept = V. "Yesterday's race" = T+V. V becomes a verb only in predicative position.

## 🎓 WORD FORMATION WORKFLOW

1. **Identify domain (C₁)** — which of the 27 active domains fits the concept?
2. **Check ontological type** — does the concept match the domain's single ontological type?
3. **Locate in matrix** — which row (V₁) and column (V₂) is the right field?
4. **Check léxico projection** — C₁ mandatory, C₂ preferential, vowels free, length by frequency
5. **Add modifier if needed** — one modifier, always last syllable
