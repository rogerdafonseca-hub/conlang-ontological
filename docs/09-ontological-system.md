# 09. Ontological System

_Version 0.8.0_

---

## Overview

The ontological system is the **deep layer** of the language — the semantic infrastructure that defines what every concept IS, independent of how it is spoken or grammatically expressed.

It answers the question: *where does this concept live in the structure of reality?*

---

## Three Distinct Systems

The language operates through three independent but synchronized systems:

| System | Layer | Purpose | Format |
|---|---|---|---|
| **Grammatical System** | Surface | Controls how things are said — tense, aspect, modality, reference, expression | CV / VC / VCV particles |
| **Ontological System** | Deep | Defines what things ARE — structural address of every concept | VCV + CV + CV... |
| **Nominal Lexicon** | Surface | Names real things — usable words for everyday speech | CVCV + CV |

These three systems are **independent** — the same consonants appear in all three with different roles. For example, *P* in the grammatical system marks prepositions; *P* in the ontological system marks the domain of matter/substance; *P* in the nominal lexicon begins any word whose primary domain is matter.

---

## The Grammatical System

The grammatical system handles **how things are communicated** between speaker and listener. It does not describe the world — it controls speech about the world.

Its building blocks are particles: short CV, VC, and VCV forms that mark tense, aspect, modality, reference, evidentiality, spatial position, expression, and more.

The grammatical system is documented in:
- `07-grammar-cv-vc.md` — CV and VC particles
- `08-expanded-grammar-vcv.md` — VCV expanded grammar
- `06-grammar-system.md` — Syntax and operator chaining

## The Ontological System

The ontological system is a **structured address space** for concepts. Every concept in the language has an ontological address — a position in a matrix defined by domain, field, and area.

**Format:** VCV + CV + CV + CV...

The opening VCV cluster is a **mode marker** — it signals "you are in the ontological system," functioning like a namespace identifier. Each subsequent CV pair narrows the address from domain to field to area.

**Scale:** Generalidade → Especificidade

The vowel scale A→U consistently encodes:
- **V₁** = position of the field within the domain (A = most general, U = most specific)
- **V₂** = position of the concept within the field

This creates a fully navigable semantic space where every concept has a unique, predictable address.

### What the ontological system is NOT

The ontological system does **not** produce spoken words. It produces addresses. The nominal lexicon then maps usable words onto those addresses.

---

## The Nominal Lexicon

The nominal lexicon is the **human interface** — the actual words speakers use. It is derived from the ontological system but follows different principles.

**Format:** CVCV + CV (extendable)

**Scale:** Frequência/importância → Raridade/tecnicidade

| Length | Use |
|---|---|
| CVCV | Universal core — most frequent and important |
| CVCVCV | Useful expansion — less daily |
| CVCVCVCV+ | Technical / specialized / rare |

### Projection Rules

The nominal lexicon projects from the ontological system following these rules:

| Element | Rule |
|---|---|
| **C₁** | MANDATORY — always mirrors the primary domain (P-words = matter, N-words = living, M-words = mental) |
| **C₂** | PREFERENTIAL — soft semantic clustering within the domain; can be overridden for phonetics |
| **Vowels** | FREE — purely distinctive; no semantic hierarchy in the lexicon |
| **Word length** | FREQUENCY — shorter = more common/important |

### Semantic Axis Law

The horizontal axis (C₂) defines **operations**, not concepts:

> **Meaning = Ontology (C₁) × Operation (C₂)**

This makes the lexicon compositional and predictable — a speaker who knows the 20 base domains can infer the semantic family of any word from its first consonant, and the operational direction from its second.

## Consonant Clusters (ECs) — Compositional System

ECs are **systematically derived** from combinations of base domains. The second element acts as an ontological operator:

| Second element | Operation | Result |
|---|---|---|
| **R** (State) | state of X | condition, phase, applied being |
| **L** (Structure) | structure of X | form, organization, pattern of X |

### ×R Series — State of X

| EC | From | Emergent Meaning |
|---|---|---|
| **PR** | P + R | Physical state / material condition |
| **BR** | B + R | Identity (entity as state) |
| **TR** | T + R | Existence / temporality |
| **GR** | G + R | Positioning state |
| **KR** | K + R | Possession (social relation as state) |
| **LR** | L + R | Configuration state |
| **MR** | M + R | Intention (mental state) |
| **XR** | X + R | Applied force |
| **FR** | F + R | Interaction state |

### ×L Series — Structure of X

| EC | From | Emergent Meaning |
|---|---|---|
| **PL** | P + L | Material structure |

| **TL** | T + L | Temporal structure / sequence system |
| **GL** | G + L | Spatial structure / geometry |
| **KL** | K + L | Social structure / institution |
| **NL** | N + L | Biological structure |

## Domain Fertility

**Highly fertile** (combine productively with most others):
- **V** — most generative; combines with all Tier 1 and most Tier 2
- **K** — social; combines with V, B, M, G, T, N, D
- **M** — mental; combines with Y, V, B, W, K, T, Ľ
- **N** — living; combines with V, K, P, G, T, M
- **P** — matter; combines with L, V, D, N, G, T
- **Y** — epistemic; combines with M, K, B, V, T

**Fertile within scope:**
- T, G, B, L, D, X — each productive within their natural domain range

**More selective:**
- R, J̌, W, Ľ, F, H — best with abstract or closely related domains

**Modal operators** (qualify fields, do not generate primary fields):
- S, Z — scalar; qualify any field they combine with

---

## Directional Fields

When two domains combine, the order determines the semantic direction:

> **A + B ≠ B + A**

- **V + N** → Biological Action Field: actions applied to living entities
- **N + V** → Behavioral Field: actions emerging from living systems

This directionality doubles the available semantic space and encodes relational meaning without additional particles.

---

## Matrix Structure

Every domain nucleus (same-domain pair, e.g. PP, MM, TT) follows a universal 5×5 matrix:

**Vertical axis (V₁):** Ontological type progression
- A = structural definition (most fundamental)
- E = fundamental unit
- I = applied state / form
- O = aggregation / organization
- U = properties / abstractions

**Horizontal axis (V₂):** Internal refinement
- A = base concept
- E = unit / refinement
- I = internal structure
- O = organization / combination
- U = specialization / definition
