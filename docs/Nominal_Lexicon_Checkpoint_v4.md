## 1. DUAL-LAYER ARCHITECTURE

The system has two synchronized layers:

| Layer | Name | Format | Function |
|---|---|---|---|
| Deep | Sistema Ontológico | VCV + CV + CV + CV... | Semantic infrastructure. No naming. Defines what something IS structurally. |
| Surface | Léxico Nominal | CVCV + CV | Human interface. Names real things. Short, usable, anchored in the world. |

**Key axiom:** The word does not carry the structure — it POINTS to it. Every word is a projection of an ontological vector. The relationship is bidirectional: word → vector, vector → word.

**Analogy:** The léxico is the view from outside the tree (leaves first). The sistema ontológico is the tree's internal structure (root to branch to tip).

## 2. SISTEMA ONTOLÓGICO

### Format
VCV + CV + CV + CV...

### VCV opener
A mode marker — signals "you are in the ontological system", same principle as numerals. May receive semantic meaning later. Currently functions as a cluster identifier only.

### Scale
Generalidade → Especificidade (A→U on both vowel axes)

### Structure
| Position | Unit | Function |
|---|---|---|
| VCV | Domain marker | Primary ontological axis |
| CV₁ | Field | First combination |
| CV₂ | Area | Refinement |
| CV₃+ | Subnucleus | Increasing specificity |

### Rules
- No naming — only structural definition
- Every position is a territory, not a description
- Depth is unlimited but most concepts live at 2–3 levels

## 3. LÉXICO NOMINAL

### Format
CVCV + CV (extendable)

### Scale
Frequência/importância → Raridade/tecnicidade

| Length | Use |
|---|---|
| CVCV | Universal core — most frequent and important concepts |
| CVCVCV | Useful expansion — important but less daily |
| CVCVCVCV+ | Technical / specialized / rare |

### Projection Rules (Léxico ← Sistema Ontológico)

| Element | Rule | Notes |
|---|---|---|
| C₁ | MANDATORY | Always mirrors the primary domain. P-words = matter, N-words = living, M-words = mental, etc. |
| C₂ | PREFERENTIAL | Soft semantic clustering within domain. Can be overridden for phonetics or collision avoidance. |
| Vowels | FREE | Purely distinctive. No semantic hierarchy in the léxico. |
| Word length | FREQUENCY | Shorter = more common/important. |

### C₂ Tendencies — Domain P (preferential, not obligatory)
| C₂ | Tendency |
|---|---|
| r | fluid, continuous |
| t | solid, rigid |
| k | granular, fragmented |
| m | organic, malleable |
| s | diffuse, gaseous |
| n | living, biological |
| l | structured, formed |

## 5. UNIVERSAL ONTOLOGICAL RULE

> A domain never contains: composition, interaction, or contextual instance.

These always go to field combinations. Specifically:
- No specific events in V → instances always involve T
- No group-in-action in K → action goes to V+K
- No emotion caused-by-X in W → causation goes to X+W

## 6. MODAL OPERATORS — NON-GENERATORS

| Domain | Name | Role |
|---|---|---|
| S | Quantity / Measurement | Quantitative modifier. Stable only in S+P and S+T |
| Z | Intensity / Degree | Scalar modifier. Z+anything = intensifier, not field |
| PL | Modal / Qualitative | Modal qualifier. Operates on fields, not a generator |

Pure nuclei (SS, ZZ, PLPL) are valid and contain the most essential scalar/modal concepts.

## 8. J̌ VS V+G — FORMAL DISTINCTION (RESOLVED)

| Criterion | J̌ | V+G |
|---|---|---|
| Ontological type | Property | Process |
| Temporal anchoring | Atemporal / general | Temporal / eventive |
| Dependency on T | No | Yes (implicit or explicit) |
| Nature | Stable pattern of motion | Change of position as event |
| Example | Orbit, current, migration (as concept) | Moving, traveling, flowing (as event) |

**Operational test:**
1. Can it exist without a specific time? → yes = J̌ / no = V+G
2. Is it something an entity HAS or something that HAPPENS? → has = J̌ / happens = V+G
3. Can you point to when it occurred? → yes = V+G / no = J̌

**Examples:**
- "The river is a current" (structural property) → J̌ | "The water is flowing now" (event) → V+G
- "Terrestrial orbit" (stable pattern) → J̌ | "The Earth is orbiting" (process) → V+G
- "Bird migration" (concept) → J̌ | "The birds migrated yesterday" → T+V+G

## 9. OPERATIONAL BOUNDARIES

### M vs W
| Example | Domain |
|---|---|
| Thinking, believing, remembering | M |
| Joy, fear, sadness | W |
| Nostalgia, anxiety | M+W |
| Intention | M (with possible W) |

Rule: W never structures thought — it only modulates. M can exist without W. W never exists without some mental state base.

### B vs Y
| Example | Domain |
|---|---|
| Sound, gesture, graphic symbol | B |
| Datum, fact, concept | Y |
| Physical text (paper with ink) | P+B |
| Knowledge in mind | M+Y |
| Silent thought | Y (no B needed) |
| Noise without meaning | B (no Y needed) |

Rule: B is the vehicle. Y is the content. They can exist independently.

### V — Process Reification
| Example | Domain |
|---|---|
| "Running" (process type) | V |
| "Yesterday's race" (instance) | T+V |
| "He ran" | V applied in grammar |

Rule: V contains process types, not instances in time.
